Estrategia Mobile-First

Escribe estilos base para el contexto más pequeño y escala hacia arriba

¿Qué significa mobile-first?

Mobile-first es una estrategia donde los estilos CSS base se escriben para pantallas pequeñas, y se usan media queries con min-width para agregar estilos conforme el viewport crece. Es lo contrario al enfoque desktop-first, que usa max-width para quitar estilos.

Desktop-first vs Mobile-first

✗ Desktop-first (menos eficiente) 
/* Base: escritorio (pesado) */
.nav {
  display: flex;
  gap: 2rem;
}

/* Sobrescribe hacia abajo */
@media (max-width: 768px) {
  .nav {
    display: none; /* deshace flex */
    flex-direction: column;
  }
}
El móvil descarga los estilos de escritorio y luego los deshace.
✓ Mobile-first (recomendado) 
/* Base: móvil (ligero) */
.nav {
  display: none;
}

/* Agrega hacia arriba */
@media (min-width: 768px) {
  .nav {
    display: flex;
    gap: 2rem;
  }
}
@media (min-width: 1024px) {
  .nav { gap: 3rem; }
}
El móvil solo carga lo que necesita. El escritorio recibe capas adicionales.

Capas de estilos mobile-first

Base (sin media query) Layout de columna única, tipografía base, colores

↓ min-width: 1024px
Escritorio Grid de 3 columnas, nav horizontal, tipografía mayor
↓ min-width: 1280px
Escritorio amplio Máx-ancho centrado, espaciado generoso
